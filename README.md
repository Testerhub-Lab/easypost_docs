# EasyPost — планировщик постов для Telegram

EasyPost — это простой и честный планировщик публикаций в Telegram: написать пост, добавить медиа, увидеть превью как в Telegram и отправить по расписанию.

**Философия EasyPost:** меньше магии — больше контроля. Меньше наворотов — больше надёжности.

---

## Для кого это приложение

- Авторы Telegram-каналов, которым нужен простой и **удобный редактор + планировщик постов** без лишних каких-то платных новоротов, которыми никогда не воспользуешься, но будешь платить.
- Команды и специалисты, которые ведут несколько каналов и хотят **единый рабочий процесс** (TODO)
- Наставники и эксперты, кто ведет наставничество через телеграм (TODO)

---

## Главная идея и ценность

### 1) Telegram-ориентированный редактор
Пост должен выглядеть в редакторе максимально близко к Telegram:
- форматирование текста
- вложения (фото/видео)
- предпросмотр
- понятное поведение альбомов (несколько фото)

### 2) Планирование — это про спокойствие
- черновики можно готовить заранее
- запланированные посты уходят сами
- статус отправки виден в интерфейсе (и ошибки тоже)

### 3) Никакой привязки “один бот - один канал”
EasyPost строится вокруг простой модели: подключил бота, подключил канал, публикуй.
(Поддерживаемые ограничения зависят от конкретных настроек Telegram и прав бота.)

---

## Как это работает под капотом (очень кратко)

- **Frontend** — интерфейс: редактор, календарь, предпросмотр, управление постами
- **Backend** — API и хранение: посты, статусы, расписание, вложения
- **Worker** — очередь отправки: по расписанию забирает задачи из очереди и публикует в Telegram

Очереди реализованы через **BullMQ + Redis**.

## Stage среда и тестирование

Stage — публичная тестовая среда, где мы проверяем основной бизнес-процесс перед релизом в прод.

**Цель:** собрать реальные баги и улучшить UX до продакшна.

### Что важно тестировать в первую очередь
**Основной бизнес-процесс:**
1) онбординг (подключение бота/канала)
2) создание поста
3) добавление медиа
4) предпросмотр
5) сохранение (черновик) или планирование
6) отправка воркером
7) корректные статусы и ошибки

**Матрица создания поста (минимальный регресс):**
- текст
- картинка
- текст + картинка
- видео
- текст + видео
- несколько картинок (альбом)
- несколько картинок + текст

## Принципы разработки EasyPost

- **Простота важнее фичей.** Сначала стабильная основа, потом расширения.
- **Наблюдаемость.** Ошибка должна быть видна и объяснима.
- **Предсказуемость.** Планирование и статусы должны работать одинаково всегда.
- **Уважение к времени пользователя.** Минимум кликов, максимум ясности.

## Roadmap (в общих чертах)

- улучшение диагностики отправки (детализация ошибок)
- шаблоны постов / повторяющиеся публикации (приоритет низкий)
- расширенные сценарии медиа
